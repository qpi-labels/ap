# 비상 대피 경로 시뮬레이션 알고리즘

## 1. 프로그램 전체 흐름 도식화

```text
[사용자 환경 설정]      [맵 데이터 생성]           [알고리즘 연산]              [결과 도출]
 대피 인원 입력   ──▶  건물 네트워크 그래프 ──▶  시뮬레이션 실행 (2가지 전략) ──▶ 대피 소요 시간 및 
 (10~200명)             (노드/간선 생성)         - 단순 최단 경로 탐색            성공 인원 비교 차트
                                               - 정체 구간 회피 탐색            화면 출력
```

---

## 2. 데이터 처리 및 함수 호출 구조

◎◎ `create_building_graph()`: 건물의 평면도(방, 복도, 출구)를 네트워크 그래프 형태로 구성하여 정의하는 함수

               ↓   

OO 데이터 입력: 건물 초기화 데이터 (노드의 위치 및 종류, 간선 연결 상태 및 이동 소요 가중치)

               ↓   

OO 명령문 블록 
(** 건물 평면도 시각화 처리: 노드 타입에 따라 색상(출구, 방, 복도)을 다르게 지정하여 맵 렌더링)

               ↓   

OO 데이터 입력: 사용자가 슬라이더를 통해 설정한 '대피 인원 수' (num_people)

               ↓   

◇◇ `run_evacuation_simulation(graph, num_people, strategy)` 호출 (경로 탐색 및 시뮬레이션 연산)

               ↓ 

OO 명령문 블록 
(** 대조군(최단 경로)과 실험군(정체 회피 경로)에 대한 연산 처리: 각 대피 인원별 이동 경로 계산, 특정 구역 인원 밀집에 따른 병목(정체) 시간 합산, 최종 소요 시간 산출)

               ↓   

△△ 결과 출력: 각 전략별 총 대피 소요 시간 및 대피 성공 인원 화면에 텍스트와 막대 차트로 출력
