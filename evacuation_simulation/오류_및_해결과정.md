# 7. 제작 과정에서 발생한 주요 오류 사항 및 해결

## 1) 발생한 오류 및 해결과정/방법 (1)
**정체 구간 회피 알고리즘(맞춤형 경로) 적용 시 다익스트라(Dijkstra) 무한 루프 및 핑퐁 현상**

<div align="center">
  <br>
  <i>(여기에 요동치는 경로 탐색이나 에러 로그 화면 캡처 이미지 삽입)</i>
  <br><br>
</div>

**[해결과정/방법 설명 1]**
* **오류 원인:** 
  실험군인 '정체 구간 회피 알고리즘'을 구현할 때, 병목 현상을 시뮬레이션하기 위해 사람이 지나간 간선(Edge)의 이동 가중치를 동적으로 증가시켰습니다. 그러나 여러 대피 인원이 동시에 이동하는 과정에서 원본 그래프의 가중치가 실시간으로 덮어씌워졌고, 이로 인해 알고리즘이 순간적으로 특정 복도의 가중치를 비정상적으로 높게 판단하여 대피 인원들이 두 노드 사이를 계속해서 왕복(핑퐁 현상)하거나, 최단 경로를 찾지 못해 무한 루프에 빠지는 치명적인 논리 오류가 발생했습니다.
* **해결 과정 및 배움:** 
  단순히 가중치를 더하는 방식으로는 복잡한 동적 네트워크를 시뮬레이션할 수 없음을 깨달았습니다. 이를 해결하기 위해 원본 그래프 속성을 직접 수정하는 대신, 시뮬레이션 내부에서 **'혼잡도 맵(Congestion Map)'**을 별도의 딕셔너리로 분리하여 관리했습니다. 또한, 시간이 지남에 따라 혼잡도가 해소되는 **'감쇠 인자(Decay Factor)'** 논리를 도입하여 인원이 빠져나가면 가중치가 다시 원래대로 복구되도록 설계했습니다. 이 과정을 통해 정적 데이터 구조와 동적 상태 데이터를 분리하여 관리하는 아키텍처 설계의 중요성과, 그래프 탐색 알고리즘의 예외 처리 방법에 대해 깊이 있게 학습할 수 있었습니다.

## 2) 발생한 오류 및 해결과정/방법 (2)
**대규모 인원 렌더링 시 SVG 애니메이션 프레임 드랍(Lag) 및 브라우저 과부하**

<div align="center">
  <br>
  <i>(여기에 브라우저 성능 분석기(DevTools)에서 프레임 드랍(빨간색 경고)이 발생한 화면 캡처 이미지 삽입)</i>
  <br><br>
</div>

**[해결과정/방법 설명 2]**
* **오류 원인:** 
  HTML/JS 기반의 프론트엔드 UI에서 시각적 효과를 극대화하기 위해 대피 인원을 나타내는 붉은색 원(SVG `<circle>`) 객체를 인원수(예: 200명 이상)만큼 생성하고, 각각에 대해 개별적인 `.animate()` 메서드를 실행했습니다. 그 결과, 브라우저가 수많은 DOM 요소의 애니메이션 궤적을 동시에 계산하고 렌더링(Repaint 및 Reflow)하면서 메모리 사용량이 급증했고 프레임 드랍(버벅거림) 현상 및 UI 멈춤 현상이 발생했습니다.
* **해결 과정 및 배움:** 
  브라우저의 렌더링 성능 한계를 인지하고 렌더링 최적화 작업에 착수했습니다. 데이터 처리 로직과 UI 시각화 로직을 분리하는 것이 핵심이었습니다. 자바스크립트 코드 내에서 `Math.min(people, 150)`과 같은 방어 코드를 추가하여 렌더링되는 최대 시각화 객체 수를 제한함으로써 DOM 과부하를 막았습니다. 또한, 시뮬레이션 소요 시간 산출(Backend/JS Logic)은 입력된 실제 인원수(200명)를 기준으로 정확히 연산하되, 화면에 보여지는 애니메이션은 시각적 이해를 돕기 위한 상징적인 표현으로 분리 적용했습니다. 이를 통해 프론트엔드 최적화 기법과 UX(사용자 경험)를 저해하지 않는 성능 타협점(Trade-off)을 찾는 실무적인 문제 해결 능력을 기를 수 있었습니다.
